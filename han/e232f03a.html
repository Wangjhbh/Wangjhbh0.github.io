如何搭建个人博客

博客生成框架 + 文件托管平台 + 站点部署服务 + 访问加速服务

博客生成框架

博客生成框架是一个应用程序，可以将用户的输入转化为网站目录文件（HTML + CSS + JS）。简单来说，博客框架可以让作为用户的我们能够以一种相对简单的方式进行博客的设置和书写，而框架负责帮助我们把这些简单的东西转化成一个完整的复杂的前端界面。

本教程使用Hexo框架进行博客的搭建，用Markdown进行博客内容的书写。

文件托管平台

可以将网站目录文件储存在云端，并提供相应的访问和修改的接口，我们使用的文件托管平台是Github，访问和修改的接口是git。

站点部署服务

将成型的网站目录文件部署到互联网上，让网络上的其他主机能够正常访问。我们使用的是github pages。国内访问速度较慢。

访问加速服务

原理是CDN加速，通过多节点的缓存来提高网络内容的解析和访问速度。我们没有使用访问加速服务。

网站目录文件的生成与上传

Hexo安装

Hexo是一个快速、简单且强大的博客框架，可以让你用最少的配置和步骤来建立一个博客网站。

由于Hexo是使用 Node.js 编写，因此在使用前需安装并配置Node.js。教程

与github建立链接

我们通过git对Github进行访问和修改，因此我们需要先安装git。教程

然后在git bash中输入下方命令生成ssh keys

ssh-keygen -t rsa -C "你注册github的邮箱地址"

设置密码后，ssh keys被保存在特定文件中（具体路径见终端输出），用记事本打开并复制ssh keys。

打开github - 右上角头像 - settings - SSH and GPG keys - New SSH key - 然后添加复制的密匙。

在git bash中输入下方命令测试ssh是否绑定成功

ssh -T git@github.com

本地创建博客

在本地新建文件夹，在文件夹中右键，用Git Bash打开。

输入下方命令初始化Hexo。

hexo init

生成本地的Hexo页面。

hexo s

打开浏览器，试着访问一下（网址见终端输出）。

在Git Bash中按Ctrl+c关闭服务器。

将初始化生成的网站目录文件上传至Github

打开github，新建一个github仓库（如：Yourname.github.io）

打开博客根目录中的_config.yml文件，更改或在末尾添加如下片段：

安装自动部署发布工具

npm install hexo-deployer-git --save

尝试生成画面

hexo g

尝试上传画面

hexo d

打开浏览器，输入https://YourRepositoryName/来尝试访问。

通过Hexo和Butterfly对博客的功能做客制化

在前边的操作中，我们已经成功的实现了博客的创建与上传，并在服务器上成功运行。在这里，我们不添加任何内容，只对博客进行定义，这些定义会体现在以后的每一篇博客中。我们需要根据Hexo文档对博客进行基本的定义，根据Butterfly文档对博客进行更细致的美观定义。您也可以只参考这篇关键定义教程来实现博客的定义。

在ssh Bash中使用如下命令来进行Butterfly 主题安装

在ssh Bash中输入下方命令来应用Butterfly主题

theme: butterfly

在ssh Bash中输入下方命令来安装必须的渲染器：

npm install hexo-renderer-pug hexo-renderer-stylus --save
